Your First Add-on

Add-ons Builder makes it easy to write, test, and build add-ons that extend the browser.

To get started, first press the Create Add-on button.

  • Create Add-on

Then Add-ons Builder will create a blank new add-on and take you to an interface for building it.

Note: if you aren't signed in yet, you'll be asked to sign in with your addons.mozilla.org account. Don't have an account? Get one!

Now, type the following code into the editing area:

  var widgets = require("widget");

  widgets.add(widgets.Widget({
    label: "Mozilla",
    image: "http://www.mozilla.org/favicon.ico"
  }));

Then click Test. You may now be asked to install the Add-ons Builder Helper extension; this is a small extension that lets Add-ons Builder integrate seamlessly with Firefox, so go ahead and install it and click Test again.

At this point, a Mozilla logo () should appear at the bottom-left corner of your browser.

Oh Noes!

Of course, you may have made a typo, in which case nothing new appears in your browser and you are confused. If this is the case, go to Firefox's Tools menu and select Error Console. Make sure that the All button appears pressed down in this window, and any errors in your code should be logged there.

How Does The Code Work?

In the code you just wrote, require is a global function that brings you functionality from another module, or reusable piece of code. It's through this mechanism that you'll access all of the browser's awesomeness, be it context menus, persistent storage, or something else entirely. In our case, we want to use the widget module, which provides a consistent, unified way for add-ons to expose their user interface.

Making It Useful

Clicking on the widget doesn't do anything, though, so let's change that by making the Mozilla website open in a new tab when the user clicks on it:

  var widgets = require("widget");

  widgets.add(widgets.Widget({
    label: "Visit The Mozilla Website",
    image: "http://www.mozilla.org/favicon.ico",
    onClick: function(event) {
      require("tabs").open("http://www.mozilla.org");
    }}));

The onClick property is an example of an event handler, which you may be familiar with if you've done any coding on the Web. Many modules in the add-ons platform reuse this convention to provide a consistent API, making learning how to do cool new things a breeze.

Checkpointing

Whenever you've reached a stopping point in your development, click the Save button. This actually saves a snapshot of your code as a new revision, and you can go back to it whenever you want by clicking the Revisions button. This allows you to play around with an idea and simply restore your code to an old revision if you decide you don't like the changes. It's just like a wiki this way: feel free to play around and don't worry about making mistakes—that's how you learn.

Making It Real

So far you've just been hacking around on Add-ons Builder website, but have no way of sharing your work with your friends and family. Fortunately, that's really easy to do: Add-ons Builder, as its name suggests, lets you create add-ons that work with the latest version of Firefox, no strings attached.

First you need to give your add-on a real name, a description, and a version number. Just click on the text "My Add-on" on the left side of the page and fill in the fields; we suggest starting at version 0.1.

Then just click Download to get the XPI file that constitutes your add-on. You can send this to friends and family, or upload it to addons.mozilla.org for all the world to see and benefit from.

Keep On Hacking!

From here, you can check out the core library documentation to discover new ways to extend the browser. Alternatively, you can browse other people's add-ons, view their source code and see how they work.

Whatever you choose, we hope you enjoy making add-ons! Don't forget to visit the Jetpack Google Group if you have any questions or want to share something with the rest of the community.
